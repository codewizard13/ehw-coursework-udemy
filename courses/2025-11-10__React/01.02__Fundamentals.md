### â–¶ï¸ L1: JSX Syntax

- JSX allows writing HTML-like markup within JavaScript, which React then transforms to `React.createElement` calls.
- It supports dynamic expressions inside braces `{}` and conditional rendering.
- Helps declarative UI construction making code more readable and maintainable.


### â–¶ï¸ L2: Creating Functional Components

- Functional components are JavaScript functions returning JSX.
- They represent reusable UI building blocks.
- Naming convention uses PascalCase, e.g., `<MyComponent />`.


### â–¶ï¸ L3: Props Basics

- Props are inputs passed from parent to child components.
- They are read-only inside the child and allow customization of components.
- Example: `<Button text="Click me" />` passes a `text` prop.


### â–¶ï¸ L4: Managing State with useState

- `useState` hook adds local state to functional components.
- Returns a state variable and a setter function.
- Triggers re-render on state change, enabling dynamic UI updates.


### â–¶ï¸ L5: Event Handling

- React uses camelCase event handlers, e.g., `onClick`.
- Functions passed to events receive the event object.
- Synthetic events normalize browser behavior for consistent interaction.


### â–¶ï¸ L6: Virtual DOM Overview

- React keeps a virtual representation of the UI in memory.
- Efficiently calculates minimal DOM updates on state/prop changes.
- Improves performance and user experience in complex applications.
